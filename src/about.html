About

Huggermuggery is a browser app for clandestine file transfers. It solves a very specific problem. It was build to bypass admin policies that disable copy-pasting files between remote desktops. It works by stuffing files into the text clipboard. If you can copy-paste text, then you can transfer files.

Instructions

  1. On each host, open a browser window to Huggermuggery.
  2. On the Sender tab
    1. Drop-and-drop the files you wish to transfer.
    2. The File Send Queue should be populated with a list of your files.
    3. Click the [Copy to Clipboard] button.
  3. On the Receiver tab (on the other host)
    1. Paste the contents of your text clipboard into the green textbox.
    2. Once you paste, you should see a list of Received Files.
    3. To save the files, click the [Save all Files] button.
      Or you can click the hyperlinks to save individual files.

Requirements

You need to use a modern browser. Huggermuggery was tested in Firefox, Chrome, and Edge. It does not work in IE11, which is a shame because on some servers that's all you get.

Troubleshooting

I wrote Huggermuggery for people with a technical skill set. As such, the solutions offered here are written with minimal explanation. They're more like hints than full solutions.

The Receiver only downloads the first file, then stops.

This happens in both Chrome and Firefox (maybe Edge too). It's a safety protocol to prevent a website from flooding your computer with files. You have to configure your browser to allow the site to download multiple files. Look for a setting called "Automatic downloads" Sometimes the browser will prompt you about the automatic download, look for a red warning icon in the address bar.

I don't have Internet access on the remote host.

I anticipated this, so I have few workarounds. They still require access to a modern web browser on the remote host.

  • If have a way to transfer files to the remote host, download Huggermuggery-portable, and copy to the host.
  • If you have access to a modern browser on the host, open about:blank, and copy-paste this code into the JS console. The code writes a copy of Huggermuggery-portable to the browser window, which you can then save locally.

What inspired me to create this?

I am a creative problem solver. I am constantly trying to find better solutions to problems.

I had a problem in which I was working with a remote host where a blanket admin policy had disabled file transfers through the OS clipboard. The only method available for transferring files was by a cumbersome and slow web portal.

I was debugging some code, which required repeated redeployment of the code to the server. For the sake of productivity, I needed a faster way to get the code onto the server. Luckily, the text clipboard was not disabled, and I was familiar with base64 encoding. I had a workaround. After getting permission from the admins (who were unwilling to enable the file clipboard, but were ok with my workaround), I wrote a quick-and-dirty WinForms app that would let me transfer the files through the text clipboard.

I wrote this browser version for a couple of reasons:

  • For the fun of it.
  • As a portfolio piece.
  • Security holes interesting topic.
  • To share the tool with others.

An apology to security admins

I suspect that some security admins have read the above with a cringe. If file transfer auditing is a part of your job, I acknowledge that by publishing Huggermuggery, I've made your job a bit more difficult. I hope that your response will not be to disable the text clipboard on remote desktop sessions, but rather to monitor it for suspicious activity. Disabling the text clipboard will kill user productivity. Also there are still workarounds for a disabled text clipboard.
